# Consent Scenarios Examples

## Purpose

This directory contains example consent scenarios demonstrating how parental consent operates across different age tiers and content types.

## Contents

Example scenarios will be added here that demonstrate:
- Opt-in consent workflows (Tier 1-2)
- Opt-out consent workflows (Tier 4)
- Acknowledgment workflows (Tier 3)
- Notification templates for parents
- Alternative provision for opted-out learners
- Consent revocation processes

## Example Scenarios

### Scenario 1: Tier 2 Puberty Education (Opt-In)

**Module:** Understanding Puberty  
**Tier:** 2 (Ages 9-12)  
**Sensitivity:** Moderate  
**Consent Type:** Opt-In Required

**Parent Notification (2 weeks before):**
```
Subject: Consent Required for Module Access: "Understanding Puberty"

Dear Parent/Guardian,

Your child will soon have the opportunity to access the module "Understanding Puberty" as part of their Tier 2 curriculum.

Topics covered:
- Physical changes during puberty
- Hygiene and self-care
- Emotional development

Sensitivity Level: Moderate
Age Tier: Tier 2 (Ages 9-12)

Deadline: March 1, 2026
```

**Outcome:** Parent provides consent → Learner accesses module → Parent receives completion summary.

### Scenario 3: Tier 4 Sexual Health Education (Opt-Out)

**Module:** Comprehensive Sexual Health  
**Tier:** 4 (Ages 16-18)  
**Sensitivity:** High  
**Consent Type:** Opt-Out Available

**Parent Notification (2 weeks before):**
```
Subject: Notification: Upcoming Module - "Comprehensive Sexual Health"

Dear Parent/Guardian,

Your child will soon have access to the module "Comprehensive Sexual Health."

Topics covered:
- Consent and healthy relationships
- Contraception and STI prevention
- Sexual orientation and identity

This module is evidence-based and non-judgmental. If you have concerns, you may opt your child out.

To opt out: [Click Here]

Deadline to opt out: March 1, 2026

If we do not hear from you, your child will access the module as part of their Tier 4 curriculum.
```

**Outcome:** Parent does not opt out → Learner accesses module → Parent receives notification of completion.

## How to Use

Developers and instructors should:
1. Identify the appropriate consent model for each module (opt-in, opt-out, acknowledgment)
2. Provide advance notification with detailed content descriptions
3. Respect parental decisions and provide alternatives for opted-out learners
4. Track consent records securely and allow revocation at any time
5. Follow up with parents post-delivery
